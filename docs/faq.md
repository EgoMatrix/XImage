## 一、安装与启动

**Q: macOS 提示「无法验证开发者」打不开？**

A: 这是 macOS 对未公证应用的默认拦截，不是病毒。

- 方法一：右键 XImage → 打开 → 在弹窗里再点「打开」。
- 方法二：「系统设置 → 隐私与安全性」滑到底部，「已阻止 XImage」旁点「仍要打开」。

**Q: Windows 提示「Windows 已保护你的电脑」？**

A: SmartScreen 误报。点「更多信息 → 仍要运行」。

**Q: Linux AppImage 双击没反应？**

A: 先 `chmod +x XImage-*.AppImage`；部分发行版需要安装 `libfuse2`：

```bash
sudo apt install libfuse2          # Debian/Ubuntu
sudo dnf install fuse-libs         # Fedora
```

**Q: 升级时旧版本要先卸载吗？**

A: 不需要。macOS 直接拖新版本覆盖；Windows 安装包会原地升级；Linux 直接换 AppImage。生成历史与配置保留。

## 二、渠道与连接

**Q: 「测试连接」失败，提示连接超时？**

A: 按顺序检查：

1. Base URL 是否拼写正确（注意 `/v1` 后缀有些渠道要、有些不要）。
2. API Key 是否过期或额度耗尽。
3. 是否需要代理。系统代理一般不会被 XImage 自动继承，请在「渠道编辑」里手动填 `http://127.0.0.1:7890` 之类。
4. 渠道服务方临时维护。

详细错误在「**设置 → 调用日志**」里。

**Q: 同一个 API Key 想配多个渠道？**

A: 完全可以，比如「OpenAI 正式 Key」「OpenAI 备用 Key」「中转 A」「中转 B」可以并存，每次生成时选用。

**Q: 中转 API 和官方 API 有什么区别？**

A: 对 XImage 来说没区别——只要对方兼容 OpenAI 接口规范、提供 Base URL 与 Key 就能用。中转一般更便宜、网络也更稳定，但请自行评估对方可信度。

**Q: 自定义 Base URL 怎么填？**

A: 填你的服务端入口的根路径，例如：
- `https://my-api.example.com/v1`
- `https://my-relay.example.com/openai/v1`

**Q: 一个渠道能用于多种能力（策划 / 生成 / 图生图）吗？**

A: 可以。同一个渠道下不同模型可以承担不同能力，XImage 会根据当前操作自动选择匹配的模型。

## 三、模型与生成

**Q: 想用的模型在「模型管理」里找不到？**

A: 用「**新增自定义模型**」，填模型 ID 即可。模型 ID 一般在渠道方文档里能查到，比如 `gpt-image-1`、`imagen-3.0-generate-001`、`stable-diffusion-xl-1024-v1-0` 等等。

**Q: 一次能生成几张？**

A: 1 ~ 4 张可调。具体取决于模型的接口能力，部分模型一次只能 1 张。

**Q: 生成很慢？**

A: 大多取决于模型与渠道，常见耗时：

- GPT-Image：5 ~ 20 秒
- Imagen：3 ~ 10 秒
- 大尺寸 / 高质量：可能 30 ~ 60 秒

如果一直卡在「生成中」超过 2 分钟，可能是网络或渠道问题，点「**终止生成**」重试，或换个渠道。

**Q: 生成失败怎么办？**

A: 预览框会变成红色「这次没生成成功」状态，点「**重试生成**」即可。原因一般在调用日志里：

- 「Insufficient balance」→ 渠道余额不足
- 「Rate limit」→ 触发限速，等几秒
- 「Invalid prompt」→ 提示词被模型拒绝（含有合规问题），换措辞
- 「Model not found」→ 模型 ID 写错了

**Q: 不同模型对中文支持差异大？**

A: 是的，XImage 不会替你做中英翻译，但 AI 策划助手在优化提示词时会按你选的模型偏好调整语言比例。

## 四、字段与提示词

**Q: 必填字段（红 \*）和选填的区别？**

A: 必填字段不填会拦截生成；选填可以全空，AI 策划会根据已填字段自动补全。

**Q: textarea 字段（可以多行的）有什么不同？**

A: 这些字段（外貌、服装、卖点、氛围、情绪等）允许多个关键词用顿号 / 逗号 / 空格分隔，AI 策划会把它们组合到提示词里。

**Q: 右侧的「chips」是什么？**

A: 一组快速点选的关键词。点击后自动追加到 textarea，连续点会拼起来。第一个带 ✨ 的会用替换模式，每点一次就换一组随机组合。

**Q: 含文字 vs 无字模式有什么区别？**

A:
- **含文字**：会把你写的标题 / 副标题 / 文本告诉模型，让它渲染在画面里。
- **无字**：让模型只画底图，文字之后你用设计软件自己加，更可控。

**Q: 「完整提示词」我可以怎么改？**

A: 三个面板：
- **创作输入**：只读。
- **优化提示词**：可改主体提示词。
- **负面提示词**：可加不想要的元素（"avoid blurry, low quality, watermark"等）。
改完点「应用修改」即可生效。

## 五、参考图

**Q: 怎么传多张参考图？**

A: 先在「**设置 → 参考图**」打开「允许多参考图」，把上限调到 2 ~ 4。回到首页就能传多张了。

**Q: 参考图会被上传到 XImage 服务器吗？**

A: **不会**。参考图只会随当前一次生成调用，发送给你设置的渠道方。XImage 没有服务器。

**Q: 想保留参考图里的人物长相？**

A: 用「**人物角色**」模式，并在外貌字段里写「保留参考图中的脸型 / 肤色 / 发型」，模型会更倾向于复用人物特征。

**Q: 参考图被忽略怎么办？**

A: 不同模型对参考图的"参考强度"不同。如果模型选成了不支持图生图的模型，参考图会被忽略——这时 XImage 应该已经把模型自动切到 remix 列表，但偶尔需要你手动确认下。

## 六、保存、迁移、隐私

**Q: 图片保存在哪里？**

A: 「**设置 → 默认保存路径**」可以指定。点「**生图记录**」里的某张图，再点「**打开文件夹**」可以直接定位。

**Q: 怎么把数据迁移到新电脑？**

A: 「**设置 → 数据存储**」里能看到 XImage 的数据目录路径。把这个目录整体复制到新电脑的相同位置即可，所有渠道、模型、历史会无缝接续。

**Q: 我的 API Key 会泄露吗？**

A: 不会。API Key 在本地加密保存，**永远**不会上传 XImage 的任何服务器（XImage 没有服务器）。Key 只会随你触发的生图请求发送给你设置的渠道方。

**Q: 生成历史能彻底清除吗？**

A: 「**设置 → 数据存储 → 清除历史**」可以一键清空数据库，但本地已保存的图片文件不会被删除（避免误删）。

**Q: XImage 会自动上传遥测吗？**

A: 默认不会。

## 七、遇到 bug 怎么办

1. 先到「**设置 → 调用日志**」复制出错信息。
2. 复现一下，记下「在哪一步、点了什么、看到了什么」。
3. 到 [Issues](https://github.com/EgoMatrix/XImage/issues) 新开一个，按模板填写。

> 报 bug 越具体修得越快，最好附：
> - 系统与 XImage 版本
> - 操作步骤（5 步以内）
> - 错误截图或调用日志片段
> - 期望行为
